# 个人内容平台需求说明书（PRD）

## 1. 项目概述

### 1.1 产品名称

个人内容平台 / 个人文章阅读平台 / 个人轻量 CMS。

### 1.2 产品定位

本产品是一个以极致阅读体验为核心的个人内容平台，面向公开访问者提供文章、博客、简历、作品、资料等内容的浏览、搜索、分享和评论能力。

产品不是传统博客首页，也不是门户网站。核心形态是：

```text
文章列表栏 | 文章目录栏 | 内容展示区
````

用户通过左侧文章标题列表或分享链接进入内容阅读；文章点击后，右侧内容区域立即展示内容，不发生整页跳转和明显加载。

### 1.3 产品目标

1. 建立一个不依赖第三方内容平台的个人内容发布阵地。
2. 支持管理员自由编辑、发布、保存个人内容。
3. 支持访客阅读、搜索、评论和分享文章。
4. 提供接近语雀、飞书、Notion 的富文本编辑体验。
5. 以桌面端阅读体验为第一优先级。
6. 支持部署到自有阿里云服务器，并通过 GitHub Actions 自动化部署。

---

## 2. 用户角色

### 2.1 管理员

管理员即网站所有者。

管理员能力：

* 登录后台
* 创建文章
* 编辑文章
* 保存文章
* 删除文章
* 上传图片
* 上传附件
* 在前台直接开启编辑模式
* 复制文章链接
* 查看文章访问量、评论数、附件数

### 2.2 普通访客

普通访客无需登录。

访客能力：

* 浏览文章
* 搜索文章
* 打开分享链接
* 查看文章目录
* 查看评论
* 发表评论
* 上传评论附件
* 回复评论

访客不能：

* 编辑文章
* 发布文章
* 删除文章
* 管理评论
* 登录后台

## 3. 信息架构

### 3.1 内容统一模型

平台不区分博客、文章、简历、作品、资料等类型。

所有内容统一作为“文章”或“内容条目”处理。

内容可以实际承载：

* 博客
* 文章
* 简历
* 作品介绍
* 资料
* 笔记
* 文档
* 其他个人内容

### 3.2 文章字段

文章基础字段：

| 字段               | 说明            |
| ---------------- | ------------- |
| id               | 系统内部唯一 ID     |
| slug             | 文章短 ID，用于 URL |
| title            | 文章标题          |
| content          | 文章正文内容        |
| rendered_content | 渲染后的正文，可选     |
| created_at       | 发布时间          |
| updated_at       | 更新时间          |
| view_count       | 访问量           |
| comment_count    | 评论数           |
| attachment_count | 附件数           |

### 3.3 文章 URL

文章 URL 使用系统自动生成短 ID。

示例：

```text
/p/a8f3k2
/p/k39d2x
/p/q1w2e3
```

规则：

* 系统自动生成 slug。
* slug 必须唯一。
* 文章标题允许重复。
* 分享链接必须可以直接打开对应文章。
* 打开文章链接后，页面展示对应文章内容。

## 4. 前台页面需求

## 4.1 页面整体布局

桌面端采用三栏布局：

```text
┌──────────────┬──────────────┬────────────────────────┐
│ 文章列表栏    │ 文章目录栏    │ 内容展示区               │
│ 搜索框        │ 当前文章目录  │ 文章正文                 │
│ 标题列表      │ 可折叠        │ 评论区                   │
└──────────────┴──────────────┴────────────────────────┘
```

第一版不考虑移动端适配。

---

## 4.2 根路径 `/`

访问根路径时，系统自动打开最新发布的文章。

规则：

* `/` 不作为传统首页。
* `/` 自动加载最新发布文章。
* 左侧仍展示文章列表和搜索框。
* 右侧展示最新文章内容。

## 4.3 文章列表栏

### 4.3.1 展示内容

左侧文章列表展示所有已发布文章。

每条文章展示：

* 标题
* 更新时间
* 访问量
* 评论数

### 4.3.2 排序规则

默认排序：

```text
最新发布在前
```

### 4.3.3 宽度调整

文章列表栏支持拖拽调整宽度。

要求：

* 默认较窄。
* 用户可以向右拉宽。
* 用户可以向左收窄。
* 需要适配长标题展示。
* 长标题不得破坏整体布局。

建议配置：

| 项目   | 建议值           |
| ---- | ------------- |
| 默认宽度 | 260px - 300px |
| 最小宽度 | 200px         |
| 最大宽度 | 560px         |

具体数值可在设计阶段微调。

## 4.4 搜索功能

### 4.4.1 搜索入口

搜索框只放在左侧文章列表顶部。

第一版不做独立搜索页。

### 4.4.2 搜索范围

搜索范围包括：

* 文章标题
* 文章正文全文

### 4.4.3 搜索排序

搜索权重：

```text
标题匹配优先
正文匹配其次
```

规则：

1. 用户输入关键词后，左侧列表只展示匹配到的文章标题。
2. 如果有标题匹配结果，标题匹配结果排在前面。
3. 如果标题没有匹配，但正文有匹配，则展示正文匹配到的文章标题。
4. 搜索后，右侧内容展示区自动展示第一个匹配结果。
5. 搜索过程不刷新页面。

### 4.4.4 搜索实现

第一版采用前端本地搜索。

要求：

* 后台保存文章后自动重建搜索索引。
* 前端加载搜索索引。
* 支持标题和正文全文搜索。
* 标题字段权重高于正文。
* 点击搜索结果后，内容区立即展示文章。

## 4.5 文章目录栏

### 4.5.1 目录位置

目录展示在文章列表栏和正文区之间。

布局：

配置：

### 4.5.3 折叠按钮

要求：

* 折叠 / 展开按钮必须醒目。
* 用户可以一键折叠目录。
* 用户可以一键重新展开目录。
* 折叠后不影响正文阅读。
* 展开后显示当前文章标题层级。

### 4.5.4 目录生成

目录根据文章标题自动生成。

支持标题层级：

点击目录项时，正文区域滚动到对应位置。

## 5. 后台管理需求

## 5.1 后台入口

后台入口：

```text
/admin
```

规则：

* 普通访客看不到编辑 / 保存按钮。
* 管理员登录后，前台显示编辑 / 保存能力。
* 管理员可通过 `/admin` 进入后台。
* 登录后直接进入文章管理列表。

---

## 5.2 管理员认证

第一版采用：

```text
管理员账号 + 密码
```

要求：

* 只支持一个管理员或少量管理员。
* 不做普通用户登录。
* 不做二次验证码。
* 不做密码找回。
* 忘记密码后，通过服务器环境变量或数据库处理。

安全要求：

* 密码必须加密存储。
* 管理接口必须校验管理员身份。
* 登录失败需要限流。
* 后台必须运行在 HTTPS 下。
* 登录状态永久保持，直到管理员手动退出。

---

## 5.3 后台首页

第一版不做仪表盘。

登录后直接进入文章管理。

后续可扩展仪表盘，展示：

* 文章数
* 总访问量
* 评论数
* 最近评论
* 最近发布

## 5.4 文章管理列表

* 标题
* 发布时间

* 编辑按钮
* 删除按钮
* 复制链接按钮

## 5.5 创建文章

管理员可以创建文章。

要求：

* 创建后进入编辑器。
* 输入标题。
* 编辑正文。
* 支持上传图片。
* 支持上传附件。
* 点击保存后立即公开。

第一版不支持：

* 草稿
* 隐藏
* 定时发布

---

## 5.6 编辑文章

管理员可编辑已有文章。

编辑入口包括：

1. 后台文章管理列表点击“编辑”。
2. 前台登录状态下，直接点击“编辑”。

前台编辑行为：

* 不跳转后台页面。
* 不弹出抽屉。
* 直接在当前文章内容区开启编辑模式。
* 编辑完成后点击保存。
* 保存成功后弹出提示。
* 保存后保持在当前编辑页，不跳转。

---

## 5.7 保存文章

保存规则：

* 保存即公开。
* 保存后覆盖原内容。
* 不保留历史版本。
* 不保留草稿。
* 不做版本恢复。

保存后行为：

* 弹出保存成功提示。
* 不跳转页面。
* 保持在当前编辑页。

---

## 5.8 删除文章

删除规则：

* 物理删除。
* 删除后不可恢复。
* 文章关联评论一起物理删除。
* 文章从搜索索引中移除。

删除确认：

* 删除前需要弹窗二次确认。
* 第一版不要求输入文章标题确认。

---

## 5.9 附件管理

第一版不做独立附件管理页。

附件只在文章编辑器中管理。

## 5.10 评论管理

## 7. 评论系统需求

## 7.1 评论基础规则

评论系统第一版规则：

* 无需登录
* 匿名评论
* 评论立即公开
* 不审核
* 不做敏感词过滤
* 不支持删除
* 不支持点赞
* 支持楼中楼回复
* 楼中楼层级不限制
* 支持文字
* 支持图片
* 支持附件
* 单条评论最多 10000 字
* 允许空文字评论，只上传附件也可以

---

## 7.2 匿名用户名

用户名规则：

* 使用中文随机名。
* 基于设备指纹永久固定。
* 同一设备长期使用同一个随机用户名。

设备识别策略：

```text
设备指纹 + localStorage + IP 组合使用
```

当设备指纹失败时：

* 使用 localStorage 匿名 ID。
* 同时结合 IP 做频率限制。
* 不因设备指纹失败而禁止评论。

---

## 7.3 评论频率限制

限制规则：

1. 每篇文章，同一设备 / IP 每天最多 5 条评论。
2. 同一设备 / IP 每 5 分钟最多 3 条评论。
3. 频率异常时出现验证码。

验证码规则：

* 正常评论不出现验证码。
* 只有频率异常时触发验证码。
* 验证码属于安全防护，不作为内容审核。

---

## 7.4 评论排序

一级评论：

```text
默认最新在前
```

后续可支持切换为最早在前。

楼中楼回复：

```text
最早在前
```

---

## 7.5 评论附件

评论附件完全开放，但必须安全处理。

单条评论附件总量：

```text
不超过 500MB
```

评论允许的附件类型与管理员附件类型一致。

包括：

* 图片
* 文档
* 压缩包
* 文本
* 音频
* 视频
* 代码文件

---

## 7.6 评论附件展示

图片展示规则：

* 小图直接展示。
* 大图显示链接。

音频 / 视频 / 其他附件展示规则：

* 评论区只显示下载链接。
* 不内嵌播放器。

管理员文章正文中的音频 / 视频：

* 可以内嵌播放。
* 可以更自由展示。

---

## 7.7 评论安全

产品原则：

```text
不做内容观点审核，但必须做技术安全防护。
```

安全要求：

* 防 XSS
* 防恶意脚本
* 防 HTML 注入
* 防接口刷写
* 防超大请求攻击
* 文件扩展名校验
* MIME 类型校验
* 文件存储隔离
* SVG、HTML、JS 等潜在危险文件不得在评论区直接执行
* 评论区附件下载需使用安全响应头
* 上传文件统一存入 OSS 或配置的存储系统

---

## 7.8 文章删除与评论删除

当文章被物理删除时：

* 文章记录删除。
* 文章评论一起物理删除。
* 文章相关评论附件是否删除，可在实现时根据文件引用关系处理。

建议：

* 若附件仅被该文章评论引用，则可一并删除。
* 若文件存在复用关系，则仅删除引用关系。

## 16. 产品风险与备注

### 16.1 部署环境风险

当前生产环境计划部署在阿里云服务器。

需要注意：

* 阿里云仍然是中心化云服务商。
* 服务器、域名、备案、网络可达性均可能受到外部环境影响。
* 如果长期目标是最大程度降低平台依赖，后续可考虑境外 VPS、对象存储镜像、多节点备份、静态镜像导出等方案。

### 16.2 删除与覆盖风险

当前确认：

* 保存直接覆盖。
* 不保留版本历史。
* 删除文章为物理删除。
* 删除评论也物理删除。

风险：

* 误操作不可恢复。
* 误删文章和评论无法通过系统恢复。

缓解方式：

* 依赖数据库每日备份。
* 删除前进行弹窗二次确认。
* 后续可增加回收站或版本历史。

### 16.3 评论开放风险

当前确认：

* 不审核。
* 不过滤敏感词。
* 评论附件完全开放。
* 评论立即公开。

风险：

* 垃圾评论
* 恶意文件
* XSS 攻击
* 刷评论
* 大文件滥用
* 法律与合规风险

缓解方式：

* 不做内容审查，但必须做安全防护。
* 严格限制频率。
* 频率异常触发验证码。
* 文件安全校验。
* 评论区不直接执行潜在危险文件。
* 必要时后续增加手动删除能力。
